# Replit Prompt — Multi-Person Activity Tracker (Mobile App)

Build a React Native (Expo) mobile app that lets a user track daily activities for multiple people (e.g., children, elderly family members, patients). The app should feel warm, calm, and caregiver-friendly — not clinical.

---

## Core Concept

This is a caregiving activity logger. The user adds **named people** (e.g., "Bader", "Daniel") and logs activities like Shower, Eating, Sleeping, Medication, etc. **for each person separately**. The user can switch between people and see each person's individual activity history and dashboard.

---

## Screens & Navigation

### 1. People Selector (Home)
- Shows a list/grid of added people as avatar cards (initials or emoji avatar + name).
- A "＋ Add Person" button to create a new person (name, optional color/avatar).
- Tapping a person navigates to their **Activity Log** screen.
- Long-press a person card to edit or delete them.

### 2. Activity Log (per person)
- Top: person's name displayed prominently.
- Toggle tabs: **"Log Activity"** | **"Dashboard"**
- **Log Activity tab:**
  - **★ FAVOURITES** section — pinned activities the user marked as favourites (long-press to favourite/unfavourite).
  - **🧍 BODY** section — grid of activity cards:
    - Shower, Bathroom, Eating, Drinking Water, Sleeping, Stretching, Exercise, Medication, Personal Care
  - **💼 PRODUCTIVITY** section — grid:
    - Work, Studying, Reading, Commuting
  - **🧠 MINDFULNESS** section — grid:
    - Meditation, Journaling, Socializing, Screen Time
  - **🏠 HOME** section — grid:
    - Cooking, Cleaning, Laundry, Groceries
  - Each activity card: rounded rectangle, soft background color unique to its category, icon, label, "long-press" hint text.
  - **Tapping** a card logs the activity with a timestamp for that person. Show a brief toast/confirmation ("✓ Shower logged for Bader at 2:05 PM").
  - **Long-pressing** a card opens options: Add to Favourites, Add Note, Set Duration, Remove from list.
  - Allow the user to **add custom activities** via a "＋ Add Activity" card at the end of any section.

- **Dashboard tab:**
  - Shows today's logged activities for this person in a timeline view (vertical list, sorted by time).
  - Summary stats: total activities today, streak info, most frequent activity this week.
  - Date picker to view past days.
  - Simple bar chart: activities per day for the last 7 days.

### 3. Settings / Manage
- Manage People (reorder, edit, delete).
- Manage Activities (add/remove/reorder activities and sections).
- App theme toggle (light/dark — default light).
- Export data as CSV.

## Design & Styling

- **Color palette:** Warm, muted tones — soft cream/beige background (#F5F1EB), muted gold for headings and active tab (#C8A96E), soft sage green for primary buttons (#7BA77B), muted rose/pink for accents.
- **Category card colors:**
  - Body: warm beige/cream
  - Productivity: soft sage/green tint
  - Mindfulness: soft lavender/purple tint
  - Home: soft peach/coral tint
- **Typography:** Rounded, friendly sans-serif (use system default or Inter/Nunito if available).
- **Cards:** Rounded corners (16px), subtle shadow, white background, centered icon + label.
- **Icons:** Use emoji or a built-in icon library (e.g., MaterialCommunityIcons from @expo/vector-icons). Each activity gets a distinct icon.
- **Tab toggle:** Pill-shaped segmented control — active tab has the sage green fill with white text; inactive is the muted rose/beige.
- **App logo area:** Leave a placeholder at the top of the People Selector screen for a logo.

## Data Model (local storage with AsyncStorage or SQLite)

```
Person {
  id: string (uuid)
  name: string
  avatar: string (emoji or initials)
  color: string (hex)
  createdAt: timestamp
}

Activity {
  id: string (uuid)
  name: string
  icon: string
  category: "body" | "productivity" | "mindfulness" | "home" | "custom"
  isFavourite: boolean (per person)
}

ActivityLog {
  id: string (uuid)
  personId: string (FK → Person)
  activityId: string (FK → Activity)
  timestamp: datetime
  duration: number (minutes, optional)
  note: string (optional)
}
```

---

## Key Behaviors

1. **First launch:** Prompt user to add their first person. Pre-populate the default activity list.
2. **Logging is one-tap:** Tap a card → instant log with current timestamp. No extra confirmation screen.
3. **Favourites are per-person:** Bader might have Medication favourited while Daniel has Exercise.
4. **Toast notifications:** Brief, non-blocking confirmation after each log.
5. **Offline-first:** All data stored locally. No backend required for v1.
6. **Smooth transitions:** Use subtle animations when switching people or tabs.
